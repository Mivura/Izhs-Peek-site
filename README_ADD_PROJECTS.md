# Инструкция по добавлению новых проектов домов

## Обзор

Сайт использует централизованную систему управления данными о домах через JSON файл. Это упрощает добавление новых проектов и делает код более поддерживаемым.

## Структура проекта

```
Izhs-Peek-site/
├── houses_data.json          # Централизованные данные о всех домах
├── app.py                     # Flask приложение
├── static/
│   ├── css/                   # Стили
│   ├── js/                    # JavaScript файлы
│   └── img/                   # Изображения
│       ├── dom1_kark/        # Папка с изображениями дома 1
│       ├── dom2_kark/        # Папка с изображениями дома 2
│       └── ...
└── templates/
    ├── base.html             # Базовый шаблон
    ├── index.html            # Главная страница
    ├── catalog.html          # Каталог домов
    ├── dom1_kark/           # Шаблон страницы дома 1
    │   └── house_1.html
    └── ...
```

## Как добавить новый проект дома

### Шаг 1: Подготовка изображений

1. Создайте папку для изображений нового дома в `static/img/`:
   ```bash
   mkdir static/img/dom6_kark
   ```

2. Поместите в эту папку:
   - Главное изображение дома (например, `1.jpg`)
   - Дополнительные фотографии (например, `2.jpg`, `3.jpg`, ...)
   - План/планировку дома (например, `plan.png`)

### Шаг 2: Создание шаблона страницы

1. Создайте папку для шаблона:
   ```bash
   mkdir templates/dom6_kark
   ```

2. Скопируйте существующий шаблон:
   ```bash
   cp templates/dom1_kark/house_1.html templates/dom6_kark/house_1.html
   ```

3. Отредактируйте шаблон:
   - Замените `dom1_kark` на `dom6_kark` в URL изображений
   - Обновите характеристики дома (площадь, этажи, спальни и т.д.)
   - Обновите комплектации и цены
   - Обновите названия планировок

### Шаг 3: Добавление данных в JSON

Откройте файл `houses_data.json` и добавьте новую запись в массив `houses`:

```json
{
  "id": 6,
  "name": "Название нового дома",
  "route": "house_kark_6",
  "template": "dom6_kark/house_1.html",
  "image_folder": "dom6_kark",
  "main_image": "dom6_kark/1.jpg",
  "description": "Описание нового дома для каталога",
  "area": "250 м²",
  "terrace": "70 м²",
  "size": "13×18 м",
  "price": "от 15 000 000 ₽",
  "characteristics": {
    "total_area": "250 м²",
    "living_area": "200 м²",
    "terrace_area": "70 м²",
    "floors": "2 этажа",
    "bedrooms": "5 спален",
    "bathrooms": "3 санузла"
  },
  "photos": [
    "/1.jpg",
    "/2.jpg",
    "/3.jpg",
    "/4.jpg"
  ],
  "plans": [
    {
      "image": "dom6_kark/plan.png",
      "title": "1 этаж"
    },
    {
      "image": "dom6_kark/plan2.png",
      "title": "2 этаж"
    }
  ]
}
```

### Шаг 4: Добавление маршрута в app.py

Откройте `app.py` и добавьте новый маршрут для дома:

```python
@app.route('/house6')
def house_kark_6():
    """Страница дома 6 с динамической загрузкой данных"""
    houses = load_houses_data()
    house = next((h for h in houses if h['id'] == 6), None)
    photos = house['photos'] if house else []
    return render_template("dom6_kark/house_1.html", photos=photos, house=house)
```

### Шаг 5: Тестирование

1. Запустите приложение:
   ```bash
   python3 app.py
   ```

2. Проверьте следующие страницы:
   - Главная страница (`/`) - должна отображать новый дом в каталоге
   - Каталог (`/catalog`) - должен включать новый дом
   - Страница дома (`/house6`) - должна корректно загружаться
   - Слайдер фотографий должен работать на мобильных устройствах

## Параметры JSON

### Обязательные поля:

- `id` (number) - уникальный идентификатор дома
- `name` (string) - название дома для отображения
- `route` (string) - имя функции маршрута в app.py (например, "house_kark_6")
- `template` (string) - путь к шаблону
- `image_folder` (string) - имя папки с изображениями
- `main_image` (string) - путь к главному изображению для каталога
- `description` (string) - краткое описание для каталога
- `area` (string) - общая площадь
- `terrace` (string) - площадь террасы
- `size` (string) - размеры дома
- `price` (string) - стоимость

### Дополнительные поля:

- `characteristics` (object) - детальные характеристики
- `photos` (array) - список путей к фотографиям (относительно папки image_folder)
- `plans` (array) - планировки этажей с заголовками

## Технологии

### Слайдер фотографий - Swiper.js

Сайт использует библиотеку Swiper.js для слайдера фотографий:
- ✅ Поддержка сенсорного управления (свайпы)
- ✅ Автопрокрутка
- ✅ Навигация стрелками и точками
- ✅ Адаптивный дизайн для мобильных устройств
- ✅ Поддержка клавиатуры

Swiper.js подключен в `templates/base.html` и автоматически доступен на всех страницах.

### Мобильная адаптация

Все страницы имеют CSS media queries для адаптации:
- Главная страница
- Каталог
- Страницы проектов
- Меню навигации
- Подвал (footer)

## Частые вопросы

**Q: Как изменить порядок домов на главной странице?**
A: Измените порядок записей в массиве `houses` в файле `houses_data.json`.

**Q: Сколько фотографий можно добавить в слайдер?**
A: Неограниченно. Добавьте пути к фотографиям в массив `photos` в JSON.

**Q: Как изменить цену дома?**
A: Измените поле `price` в JSON и соответствующие цены в шаблоне дома.

**Q: Нужно ли перезапускать приложение после изменения JSON?**
A: Да, после изменения `houses_data.json` нужно перезапустить Flask приложение.

## Поддержка

При возникновении проблем:
1. Проверьте консоль Flask на наличие ошибок
2. Убедитесь, что все пути к файлам указаны правильно
3. Проверьте валидность JSON (можно использовать jsonlint.com)
4. Убедитесь, что имя функции маршрута в app.py совпадает с полем `route` в JSON

## Резервное копирование

Перед внесением изменений рекомендуется создать резервную копию:
```bash
cp houses_data.json houses_data.json.backup
cp app.py app.py.backup
```
